Pour faciliter l’installation des outils de développement sur votre ordinateur, et garantir une homogénéité des environnements de travail, nous fournissons une image de machine virtuelle équipée d’un système d’exploitation Linux minimal.

Des explications complémentaires sont disponibles en annexe, si vous souhaitez installer les logiciels vous-même sans utiliser la machine virtuelle fournie.

Dans ce cas, vous devez également télécharger les fichiers sources de base de cette activité.

Pour la plupart d’entre vous, les instructions ci-dessous suffisent.

Installation

  1. Installez le logiciel VirtualBox, de préférence en version 6.0. Téléchargez également le VirtualBox Extension Pack avec le même numéro de version que VirtualBox. Le fichier Extension pack (.vbox-extpack) s’installe en ouvrant VirtualBox et en allant dans Outils → Paramètres → Extensions.
  2. Téléchargez l’image de la machine virtuelle CoCiNum.ova :
    • sur le site SharePoint de l’ESEO
    • ou sur Google Drive
    • ou en utilisant un client BitTorrent : CoCiNum.ova.torrent
  3. Ouvrez VirtualBox.
    • Dans le menu Fichiers, choisissez Importer un appareil virtuel.
    • Choisissez le fichier CoCiNum.ova que vous avez téléchargé.
    • Pressez les boutons Suivant, puis Importer.

Démarrage et test de la machine virtuelle

  1. Dans VirtualBox, sélectionnez la machine CoCiNum et pressez le bouton Démarrer.
  2. Le démarrage complet peut prendre plusieurs minutes selon le PC que vous utilisez. Vous devez obtenir une fenêtre CoCiNum affichant un environnement de bureau avec un terminal ouvert.

  1. Pour pouvoir utiliser l’action Copier-Coller entre la machine virtuelle et votre environnement de travail principal, allez dans le menu Périphériques et choisissez Presse-papier partagé, puis Bidirectionnel (remarque : sous Linux, dans le terminal, l’action Coller se fait avec la combinaison de touches Ctrl-Shift-V).
  2. Dans le terminal, saisissez les commandes suivantes :
cd $HOME/CoCiNum/ghdl-demo/tests
make
  1. Vous devez voir s’afficher une fenêtre contenant un bouton « Click Here » et un disque gris en-dessous.
  2. Pressez le bouton : le disque doit devenir vert.
  3. Refermez la fenêtre.

La commande make ci-dessus démarre la simulation d’une entité VHDL. Le simulateur est associé à une interface graphique simple écrite en Python : les appuis sur le bouton mettent à '1' une entrée de l’entité ; la couleur du disque correspond à l’état d’une sortie de l’entité.

Dépannage de la machine virtuelle

Échec de l’importation de l’appareil virtuel

Si vous obtenez ce message avec un code d’erreur E_INVALIDARG (0x80BB0004), il est possible que votre fichier CoCiNum.ova soit incomplet ou corrompu, peut-être à cause d’une interruption du téléchargement.

Si vous obtenez ce message avec un code d’erreur E_INVALIDARG (0x80070057), vous pouvez tenter la procédure suivante :

  1. Décompresser le fichier CoCiNum.ova en utilisant un logiciel comme 7-zip, par exemple. Vous devez obtenir un fichier CociNum_disk_001.vmdk. Si la décompression échoue, il est possible que votre fichier CoCiNum.ova soit incomplet ou corrompu, peut-être à cause d’une interruption du téléchargement.
  2. Dans VirtualBox, créez une nouvelle machine virtuelle avec les caractéristiques suivantes :
    • Nom : CoCiNum
    • Type : Linux
    • Version : Debian (32-bit)
    • Taille de la mémoire : 1024 Mio
    • Utiliser un fichier de disque dur virtuel existant : choisir le fichier CociNum_disk_001.vmdk.

Échec de l’ouverture de session

Si vous obtenez le message d’erreur « Implementation of the USB 2.0 controller not found! », cela peut signifier deux choses :

  • Le pack d’extensions VirtualBox n’est pas installé.
  • Le pack d’extensions n’a pas le même numéro de version que votre installation de VirtualBox.

Vous pouvez résoudre ce problème de deux manières :

  • En installant la bonne version du pack d’extensions de VirtualBox.
  • Ou en utilisant un contrôleur USB 1.1 dans la configuration de votre machine virtuelle.

Not in a hypervisor partition, VT-x/AMD-V is disabled in the BIOS

L’exécution de la machine virtuelle nécessite le support matériel de la virtualisation sur votre PC.

Si vous obtenez l’erreur « Not in a hypervisor partition », suivie de « VT-x is disabled in the BIOS », ou « AMD-V is disabled in the BIOS », vous devez redémarrer votre PC et entrer dans la configuration du BIOS pour activer le support de la virtualisation.

La procédure exacte varie selon les modèles de PC et il est difficile de donner plus de détails. La technologie de virtualisation à activer se nomme :

  • VT-x pour les processeurs Intel,
  • AMD-V pour les processeurs AMD.

Si vous ne pouvez vraiment pas activer cette option, vous devez modifier la configuration de la machine virtuelle pour désactiver l’accélération (Configuration → Système → Accélération).

Dans VirtualBox 6.1, on ne peut pas désactiver l’accélération. Vous devez utiliser une version antérieure.

Le système d’exploitation de la machine virtuelle ne démarre pas complètement

Différents problèmes peuvent se présenter :

  • Un message « Fixing recursive fault but reboot is needed ».
  • Un démarrage qui s’arrête avec un message du type « /dev/sda1: clean, …/… files …/… blocks ».

Les personnes qui ont eu ces problèmes ont pu les résoudre de différentes manières :

  • En choisissant Debian (64-bit) au lieu de Debian (32-bit) dans la configuration de la machine virtuelle.
  • En modifiant le nombre de cœurs dans la section Système → Processeur (2 au lieu de 1), et la quantité de mémoire vidéo dans la section Affichage (128 Mo).
  • En changeant de PC.
